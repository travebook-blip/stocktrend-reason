# 自動更新線上看板 — 部署指南

目標：把這個籌碼控盤看板放上網，**每天收盤後自動更新**，你只要維護一份 `watchlist.txt`。
架構：GitHub Actions（每天排程）→ 用 FinMind 抓真實資料 → `stock_analyzer.py` 算好全部指標 → 產生 `data.js` → GitHub Pages 顯示 `index.html`。全程免費、無需自己的伺服器。

---

## 你會用到的檔案
放進同一個 GitHub repo 的根目錄：
- `index.html`　　　看板網頁（讀 `data.js` 顯示）
- `stock_analyzer.py` / `data_sources.py`　分析引擎
- `requirements.txt`　Python 套件
- `watchlist.txt`　　你的股票清單（改這份就好）
- `update.yml` → 要放在 **`.github/workflows/update.yml`**（注意路徑）

`data.js` / `data.json` 由排程自動產生，不用自己建。

---

## 步驟一：拿 FinMind 免費 token
1. 到 https://finmindtrade.com/ 註冊登入。
2. 會員中心 → 複製你的 API token。
（免費額度足夠每天更新數十檔。）

## 步驟二：建立 GitHub repo 並上傳檔案
1. GitHub 右上 ➕ → New repository，取個名字（例如 `stock-radar`），設 **Public**（Pages 免費）。
2. 上傳上面那些檔案。`update.yml` 要放到 `.github/workflows/` 資料夾下（路徑必須是 `.github/workflows/update.yml`）。
   - 用網頁上傳：Add file → Create new file → 檔名直接打 `.github/workflows/update.yml`，貼上內容即可自動建立資料夾。

## 步驟三：把 token 存成 Secret
Repo → **Settings → Secrets and variables → Actions → New repository secret**
- Name：`FINMIND_TOKEN`
- Secret：貼上你的 FinMind token → Add secret

## 步驟四：開啟 GitHub Pages
Repo → **Settings → Pages**
- Source 選 **Deploy from a branch**
- Branch 選 **main**、資料夾選 **/(root)** → Save
- 稍候會出現你的網址：`https://<你的帳號>.github.io/<repo名>/`

## 步驟五：先手動跑一次
Repo → **Actions** 分頁 → 左側「每日更新籌碼控盤資料」→ 右側 **Run workflow** → Run。
- 約 1~2 分鐘跑完，成功後 repo 會多出 `data.js` / `data.json`。
- 打開你的 Pages 網址，就會看到「即時資料 · 更新時間」與你清單裡每檔的完整分析。

之後**每個交易日台灣時間 18:00 會自動更新**，你不用做任何事。

---

## 日常使用
- 要換股票：編輯 `watchlist.txt`（每行一檔代號）→ commit。下次排程就會用新清單；想立刻更新就到 Actions 手動 Run 一次。
- 看板會自動標出：三線多排、K>D、KD/MACD 近 3 日金叉死叉、MACD 紅綠柱、月線扣低/扣高，以及外資/投信買賣張數與「控盤天平」（漲是誰買上去、跌是誰賣下來、目前誰控盤）。

## 常見問題
- **Actions 失敗、紅色 X**：點進去看 log。最常見是 `FINMIND_TOKEN` 沒設或打錯 → 重設 Secret。
- **某檔沒資料**：可能是新上市/代號錯/當天 FinMind 尚未更新；其他檔不受影響。
- **想改更新時間**：編輯 `update.yml` 裡的 `cron`（用 UTC，台灣時間需 −8 小時）。
- **想本機先測**：`pip install -r requirements.txt` 後跑 `python stock_analyzer.py --live --token 你的TOKEN`，會在本地產生 `data.js`，用瀏覽器開 `index.html` 即可預覽。不加 `--live` 則用內建範例資料。
